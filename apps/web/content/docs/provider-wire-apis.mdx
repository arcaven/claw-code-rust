## Wire API Values

Supported `wire_apis` and `invocation_method` values:

| Value | API surface |
| --- | --- |
| `openai_chat_completions` | OpenAI-compatible `/v1/chat/completions`. |
| `openai_responses` | OpenAI-compatible `/v1/responses`. |
| `anthropic_messages` | Anthropic-compatible `/v1/messages`. |

If a provider declares `wire_apis`, a binding can only use an
`invocation_method` listed by that provider.

When a binding omits `invocation_method`, Devo defaults to
`openai_chat_completions`.

## Example

```toml
[providers.main]
enabled = true
name = "Main Provider"
base_url = "https://api.example.com/v1"
credential = "main_api_key"
wire_apis = ["openai_responses"]

[model_bindings.gpt-main]
enabled = true
model_slug = "kimi-k2.5"
provider = "main"
model_name = "kimi-k2.5"
invocation_method = "openai_responses"
```
