## Wire API 值

支持的 `wire_apis` 和 `invocation_method` 值：

| 值 | API 表面 |
| --- | --- |
| `openai_chat_completions` | 兼容 OpenAI 的 `/v1/chat/completions`。 |
| `openai_responses` | 兼容 OpenAI 的 `/v1/responses`。 |
| `anthropic_messages` | 兼容 Anthropic 的 `/v1/messages`。 |

如果提供方声明了 `wire_apis`，模型绑定只能使用该提供方列表中的 `invocation_method`。

如果模型绑定省略 `invocation_method`，Devo 默认使用 `openai_chat_completions`。

## 示例

```toml
[providers.main]
enabled = true
name = "Main Provider"
base_url = "https://api.example.com/v1"
credential = "main_api_key"
wire_apis = ["openai_responses"]

[model_bindings.gpt-main]
enabled = true
model_slug = "kimi-k2.5"
provider = "main"
model_name = "kimi-k2.5"
invocation_method = "openai_responses"
```
