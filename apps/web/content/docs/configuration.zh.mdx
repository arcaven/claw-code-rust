## 配置文件

用户级配置存储在：

```text
<DEVO_HOME>/config.toml
```

`DEVO_HOME` 默认是 `~/.devo`。

工作区级配置存储在：

```text
<workspace>/.devo/config.toml
```

凭证材料单独存储在：

```text
<DEVO_HOME>/auth.json
```

工作区配置会按字段覆盖用户配置。如果同一个提供方或模型绑定同时存在于两个文件中，工作区字段优先；只存在于用户配置中的字段仍然有效。

## 提供方配置

提供方是可复用的连接记录。它包含提供方身份、连接 URL、凭证引用、支持的 Wire API，以及启用状态。

```toml
[providers.openrouter]
enabled = true
name = "OpenRouter"
base_url = "https://openrouter.ai/api/v1"
credential = "openrouter_api_key"
wire_apis = ["openai_chat_completions", "openai_responses"]
```

`credential` 字段是指向 `auth.json` 的凭证 id；API key 不应直接存储在 `config.toml` 中。

```json
{
  "version": 1,
  "credentials": {
    "openrouter_api_key": {
      "kind": "api_key",
      "value": "sk-or-example"
    }
  }
}
```

当前支持的凭证类型：

| 值 | 含义 |
| --- | --- |
| `api_key` | 提供方适配器使用的 API key 或 bearer token。 |

## 模型绑定

模型绑定会把一个支持的模型和一个配置好的提供方连接起来，使其成为可调用模型。

```toml
[defaults]
model_binding = "deepseek-v4-pro-openrouter"

[model_bindings.deepseek-v4-pro-openrouter]
enabled = true
model_slug = "deepseek-v4-pro"
provider = "openrouter"
model_name = "deepseek/deepseek-v4-pro"
display_name = "DeepSeek V4 Pro"
invocation_method = "openai_chat_completions"
default_reasoning_effort = "high"
```

重要字段：

| 字段 | 含义 |
| --- | --- |
| `model_slug` | Devo 支持模型的稳定 id。 |
| `provider` | 来自 `[providers.<id>]` 的提供方 id。 |
| `model_name` | 发送到提供方的模型名称。 |
| `display_name` | 客户端中显示的用户可读标签。 |
| `invocation_method` | 请求使用的提供方 Wire API。 |
| `default_reasoning_effort` | 支持推理的模型可使用的默认推理选择。 |
