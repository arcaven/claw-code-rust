## Configuration Files

User-level configuration is stored in:

```text
<DEVO_HOME>/config.toml
```

`DEVO_HOME` defaults to `~/.devo`.

Workspace-level configuration is stored in:

```text
<workspace>/.devo/config.toml
```

Credential material is stored separately in:

```text
<DEVO_HOME>/auth.json
```

Workspace configuration overlays user configuration field by field. If the same
provider or model binding exists in both files, workspace fields take
precedence, while user-only fields remain effective.

## Provider Config

Providers are reusable connection records. They contain provider identity,
connection URL, credential reference, supported wire APIs, and enabled state.

```toml
[providers.openrouter]
enabled = true
name = "OpenRouter"
base_url = "https://openrouter.ai/api/v1"
credential = "openrouter_api_key"
wire_apis = ["openai_chat_completions", "openai_responses"]
```

The `credential` field is an id that points to `auth.json`; API keys should not
be stored directly in `config.toml`.

```json
{
  "version": 1,
  "credentials": {
    "openrouter_api_key": {
      "kind": "api_key",
      "value": "sk-or-example"
    }
  }
}
```

Currently supported credential kind:

| Value | Meaning |
| --- | --- |
| `api_key` | API key or bearer token used by a provider adapter. |

## Model Bindings

A model binding makes one supported model invocable through one configured
provider.

```toml
[defaults]
model_binding = "deepseek-v4-pro-openrouter"

[model_bindings.deepseek-v4-pro-openrouter]
enabled = true
model_slug = "deepseek-v4-pro"
provider = "openrouter"
model_name = "deepseek/deepseek-v4-pro"
display_name = "DeepSeek V4 Pro"
invocation_method = "openai_chat_completions"
default_reasoning_effort = "high"
```

Important fields:

| Field | Meaning |
| --- | --- |
| `model_slug` | Stable Devo-supported model id. |
| `provider` | Provider id from `[providers.<id>]`. |
| `model_name` | Provider-specific model name sent on the wire. |
| `display_name` | User-facing label shown in clients. |
| `invocation_method` | Provider wire API used for requests. |
| `default_reasoning_effort` | Optional default reasoning selection for capable models. |
