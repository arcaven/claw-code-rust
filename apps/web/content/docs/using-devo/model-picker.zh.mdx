---
title: Model Picker
description: 在已配置的 model bindings 之间切换。
---

使用 `/model` 切换当前 session 的 active model。

```text
/model
```

Picker 会在底部 pane 打开。使用 `Up` 和 `Down` 移动，`Enter` 选择，`Esc` 关闭且不修改模型。

## Picker 显示什么

Picker 来自保存的 model bindings。每个条目显示 display name，并在可用时在右侧显示 provider hint。

当前模型会在列表中标记。如果配置的 binding 有 display name，Devo 使用该 display name；否则回退到 catalog model display name 或原始 model slug。

## 直接选择

也可以直接传入 model 或 binding name：

```text
/model gpt-5-codex
```

当参数匹配已保存 model binding 时，Devo 使用该 binding，包括 provider、request model name、wire API 和 display name。当它匹配 catalog model slug 时，Devo 应用该 catalog model。如果两者都不匹配，Devo 仍会把请求的 model name 设置到 turn context。

## Reasoning Effort

某些模型暴露 configurable reasoning effort。当你选择这些模型时，Devo 可能先打开第二个 picker 选择 reasoning effort，再完成模型切换。

Reasoning effort picker 使用模型配置的 capability 和 reasoning-effort options。如果模型没有暴露选项，Devo 会直接应用模型。

选择后，Devo 更新 header，并把 model 和 reasoning effort selection 作为 turn context 发送给 runtime。

## Session Scope

`/model` 修改 active session context。它不会改写 provider 配置文件。

添加新的 provider 或 model binding：

```bash
devo onboard
```

如果希望某个 binding 成为启动默认值，应更新 provider configuration，而不是依赖 session 内的 `/model` 切换。

## Devo 工作中

Devo 生成任务时会阻止模型切换。如果在 running turn 中尝试切换模型，transcript 会提示生成期间无法修改 model。

等待当前 turn 完成，用 `Esc` 中断，或开始新 session 后再切换。
