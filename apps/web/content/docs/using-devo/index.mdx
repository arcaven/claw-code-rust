## Main Loop

Most Devo work follows this loop:

1. Open a workspace with `devo`.
2. Type a request in the composer.
3. Review Devo's plan, tool activity, or questions in the transcript.
4. Approve or deny tool requests when Devo asks.
5. Continue with another prompt, switch modes, or exit and resume later.

The composer is not only a chat input. It also supports slash commands, model
selection, file references, shell mode, and plan-mode toggling.

## Common Entry Points

| Task | Use |
| --- | --- |
| Configure Devo again | `devo onboard` |
| Switch model | `/model` |
| Show session status | `/status` |
| Clear the visible transcript | `/clear` |
| Compact long context | `/compact` |
| Start a fresh session | `/new` |
| Browse saved sessions | `/resume` |
| Change permissions | `/permissions` |
| List MCP servers | `/mcp` |
| List skills | `/skills` |
| Exit | `/exit` |

## Modes And Triggers

`Shift+Tab` switches between Build mode and Plan Mode. Build mode lets Devo
make changes subject to permissions. Plan Mode is for investigation and proposed
plans before file edits.

Use `@` inside the composer to search for files, skills, and MCP resources.
Use `!` at the start of the composer to run shell commands through Devo's shell
surface.

Use `/goal` when the work has a larger end state that should persist across
turns.
