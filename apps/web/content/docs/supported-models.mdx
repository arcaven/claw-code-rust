## Supported Model Definitions

Supported model definitions are built into Devo. They describe intrinsic model
capabilities and do not contain API keys, provider URLs, or provider-specific
model names.

| Model slug | Display name | Channel | Default wire API | Thinking | Input |
| --- | --- | --- | --- | --- | --- |
| `qwen3-coder-next` | qwen3-coder-next | Qwen | `openai_chat_completions` | unsupported | text |
| `kimi-k2.5` | kimi-k2.5 | Kimi | `openai_chat_completions` | toggle | text, image |
| `minimax-m2.7` | minimax-m2.7 | MiniMax | `openai_chat_completions` | unsupported | text |
| `glm-5.1` | glm-5.1 | GLM | `openai_chat_completions` | toggle | text |
| `deepseek-v4-flash` | deepseek-v4-flash | DeepSeek | `openai_chat_completions` | toggle, `high`/`max` | text |
| `deepseek-v4-pro` | deepseek-v4-pro | DeepSeek | `openai_chat_completions` | toggle, `high`/`max` | text |

If you would like to contribute a new model, open an issue in the
[GitHub repository](https://github.com/7df-lab/devo).

## Boundary

Supported model definitions describe the model itself. A model only becomes
invocable after configuration binds it to a provider and a provider-specific
model name.
