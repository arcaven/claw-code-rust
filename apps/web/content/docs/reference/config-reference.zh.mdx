本页列出主要 config files 和 fields。引导式配置请先读[配置基础](../configuration/config-basics)、[提供商配置](../configuration/provider-setup) 和[模型绑定](../configuration/model-bindings)。

## 文件

| File | Purpose |
| --- | --- |
| `<DEVO_HOME>/config.toml` | User-level Devo config。 |
| `<DEVO_HOME>/auth.json` | User-level secrets 和 credentials。 |
| `<workspace>/.devo/config.toml` | Workspace-level config overlay。 |

默认 `DEVO_HOME` 路径：

| Platform | Default |
| --- | --- |
| macOS/Linux | `~/.devo` |
| Windows | `C:/Users/your-username/.devo` |

如果设置了 `DEVO_HOME`，它必须指向已经存在的目录。

## 优先级

Effective config 按以下顺序解析，优先级从高到低：

1. command-line startup arguments；
2. `<workspace>/.devo/config.toml`；
3. `<DEVO_HOME>/config.toml`；
4. built-in defaults。

`auth.json` 是 user-scoped，不应提交到仓库。

## `[server]`

| Field | Type | Meaning |
| --- | --- | --- |
| `listen` | string array | Server listen endpoints。 |
| `max_connections` | integer | 最大 concurrent client connections。默认 `32`。 |
| `event_buffer_size` | integer | 每个 connection 的 event buffer capacity。默认 `1024`。 |
| `idle_session_timeout_secs` | integer | Idle session timeout 秒数。默认 `1800`。 |
| `persist_ephemeral_sessions` | bool | 是否持久化 ephemeral sessions。默认 `false`。 |

### `[server.auth]`

| Field | Type | Meaning |
| --- | --- | --- |
| `enabled` | bool | 非 initialization server methods 是否需要 server authentication。默认 `false`。 |
| `method_id` | string | Initialization 时 advertised 的 server authentication method id。默认 `agent-login`。 |
| `name` | string | 用户可读 authentication method name。默认 `Agent login`。 |
| `description` | string | 可选 authentication method description。 |
| `logout` | bool | Auth enabled 时是否 advertise 并支持 ACP `logout`。默认 `true`。 |

启用后，auth state 按 connection 隔离，并同时 gate ACP methods 和 legacy
Devo server methods。

## `auth.json`

`auth.json` 存储 user-scoped credentials：

```json
{
  "version": 1,
  "credentials": {
    "deepseek_api_key": {
      "kind": "api_key",
      "value": "sk-your-key"
    }
  }
}
```

Credential fields：

| Field | Type | Meaning |
| --- | --- | --- |
| `version` | integer | Credential file schema version。当前值为 `1`。 |
| `credentials.<id>.kind` | `api_key` | Credential kind。 |
| `credentials.<id>.value` | string | Secret value。 |

不要提交 `auth.json`。

## Update Checks

默认 update config 启用 startup checks，最小间隔 24 小时：

```toml
[updates]
enabled = true
check_on_startup = true
check_interval_hours = 24
```

需要完全离线运行的环境应禁用 startup checks。
