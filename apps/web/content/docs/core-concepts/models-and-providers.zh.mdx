---
title: Models and Providers
description: 理解 model definitions、providers 和 model bindings。
---

Devo 将 model capability metadata 与 provider connection details 分开。因此同一个 Devo UI 可以接 OpenAI、Anthropic、DeepSeek、OpenRouter、本地 gateways 和其他兼容 providers。

## 三层结构

| Layer | 描述什么 | 包含 secrets？ |
| --- | --- | --- |
| Model profile | 模型能力、display name、context window、reasoning support、modalities 和 max output tokens。 | 否 |
| Provider vendor | 配置的 provider endpoint、base URL、credential reference、headers 和 supported wire APIs。 | 仅 credential reference |
| Model binding | model slug 与 provider vendor 的连接，包括 provider model name、display name、wire API 和 default reasoning effort。 | 否 |

Model profiles 不应包含 API keys、provider URLs 或 invocation methods。

Provider vendors 描述如何访问一个服务。Model bindings 描述 Devo model slug 应使用哪个 provider 和模型名称。

## Model Profile

Model profile 是 Devo 用于渲染 UI 和塑造请求的 capability record，包括：

- stable model slug；
- display name；
- base instructions；
- context window 和有效可用 context；
- reasoning capability；
- default reasoning effort；
- tool-use support；
- image support；
- streaming support；
- prompt-caching support；
- max output tokens。

Model picker 和 status line 使用这些 metadata。Compaction 和 context status 也依赖 context window 字段。

## Provider Vendor

Provider vendor 是 connection record，包括：

- provider id 或 name；
- optional base URL；
- credential id；
- optional headers；
- enabled state；
- 一个或多个 supported wire APIs。

API key 本身属于 user auth storage，不属于 model profile。Onboarding 会为你写入 credential reference 和 provider settings。

## Model Binding

Model binding 将 model profile 与 provider vendor 连接起来。它回答：

- 哪个 provider 服务这个模型？
- 发给该 provider 的 model name 是什么？
- Devo 应使用哪个 wire API？
- UI 应显示什么 display name？
- 应选择什么 default reasoning effort？

例如，DeepSeek binding 可以使用 `deepseek-v4-flash` 这样的 Devo model slug，发送 DeepSeek 要求的 provider model name，并通过 DeepSeek Anthropic 兼容 base URL 使用 `anthropic_messages` wire API。

## Wire APIs

Devo 当前建模这些 provider wire APIs：

| Wire API | 含义 |
| --- | --- |
| `openai_chat_completions` | OpenAI 兼容 chat completions requests。 |
| `openai_responses` | OpenAI 兼容 Responses API requests。 |
| `anthropic_messages` | Anthropic 兼容 Messages API requests。 |

选择与 endpoint 匹配的 wire API，而不是只看公司名称。对于 DeepSeek 官方 API，Devo 推荐 Anthropic 兼容路径，因为 hosted `web_search` 和 `web_fetch` 可通过该 provider route 使用。

## Resolution

启动时和一个 turn 开始时，Devo 会 resolve active model binding。Runtime 会组合：

- selected model slug；
- configured binding；
- provider vendor；
- `/model` 带来的 session 或 turn override；
- selected thinking 或 reasoning mode。

结果就是 Devo 为该 turn 发送的具体 provider request。

如果未配置 provider，Devo 无法运行 model requests，并会引导你运行：

```bash
devo onboard
```
